# Privacy And Security

Social triage systems touch sensitive data. Treat the public repo as a template only.

## Never Commit

- Passwords
- OAuth tokens
- Browser cookies
- Session exports
- Real social handles
- Real group URLs
- Real phone numbers
- Employee names
- Customer names
- Private message screenshots
- Legal complaints
- Real decision inbox exports

## Authentication

Authenticate through approved local flows only.

Do not paste passwords, one-time codes, recovery codes, cookies, or API secrets into chat.

If a platform asks for a code or identity check, the human should complete it directly.

## Urgent Notifications

Urgent notifications should contain pointers, not full sensitive threads.

Recommended:

```text
Triage ID
Priority
One-line brief
Notion page URL
```

Avoid putting full complaint text, legal allegations, payment details, or personal information into SMS/chat.

## Action Safety

Prefer review for ambiguous actions.

Be careful with:

- Sending replies
- Deleting comments
- Banning users
- Accepting ambiguous requests
- Handling legal or payment disputes
- Handling personal reach-outs

Low-risk actions should still be reversible where possible.
