# Notion Decision Inbox Schema

Create a Notion database named `Executive Decision Inbox`.

## Properties

| Property | Type | Purpose |
| --- | --- | --- |
| Thread / Decision | Title | Short decision title prefixed with the exact social source, such as `Facebook Messenger:` or `LinkedIn Message:` |
| Triage ID | Text | Stable ID generated by the agent |
| Status | Select | Waiting for Executive, Executive Responded, Ready to Act, Archive, Handled at Source, Done, Snoozed |
| Priority | Select | Urgent, High, Normal, Low |
| Source Channel | Select | Facebook Messenger, LinkedIn Message, LinkedIn Connection Request, Facebook Friend Request, Facebook Group, Other |
| Source Thread Link | URL | Profile, post, group, thread, or message URL |
| Observed Sent At | Date | Timestamp observed on the source platform |
| From / Actor | Text | Sender, commenter, requester, or member |
| All Parties | Text | People or organizations involved |
| Rule Category | Relation | Optional relation to Social Triage Rules |
| Social Action | Select | Reply on Social, Delegate, Email/Text Delegate, Call/Text, Need More Info, No Social Reply, Archive / Ignore, Schedule |
| Exact Ask | Text | What the person is asking or saying |
| Suggested Reply / Action | Text | Agent recommendation |
| Summary | Text | Plain-language context |
| Why It Matters | Text | Why the executive should care |
| Assistant Notes | Text | Links, confidence, screenshots, source rule, automation notes |
| Completed Date | Date | Filled when done |
| Last Checked At | Date | Filled by responder automation |

## Decision Item Requirements

Every decision item should let the executive decide without opening the social platform.

Include:

- Source-specific title prefix, such as `Facebook Messenger:`, `LinkedIn Message:`, `LinkedIn Connection Request:`, `Facebook Group:`, or `Facebook Friend Request:`
- Source channel and source thread link
- Observed sent/activity timestamp
- Sender or actor
- Full relevant inbound message, post, or comment text
- Exact ask
- Suggested action
- Summary
- Why it matters
- Rule match and confidence

The page body must not be blank. It should include:

1. `Decision Brief`
2. `Exact Ask`
3. `Suggested Action`
4. `Context`
5. `Why It Matters`
6. `Source And Meta`
7. `Run Notes`

For Messenger, LinkedIn DM, and other direct-message items, include the full relevant inbound message transcript in the page body with sender names, observed timestamps, and line breaks when available.

For group posts or comments, include the full relevant post/comment text, author, observed timestamp, and source link when available.

After creating or updating a decision item, fetch it back and verify that the page body is not blank before reporting the run as complete.

For urgent items, create the Notion item first, then send the compact urgent notification with the Notion page URL.

## Response Handling

When the executive comments in Notion or changes `Status`, the responder should use `Source Channel` and `Source Thread Link` to return to the original source.

Route examples:

- `Facebook Messenger` -> linked Messenger conversation
- `LinkedIn Message` -> linked LinkedIn conversation
- `LinkedIn Connection Request` -> linked request
- `Facebook Group` -> linked post, comment, member request, or moderation item

Do not assume all follow-up actions are email actions. If the source link or authentication is missing, keep the item active and add a blocker note.
